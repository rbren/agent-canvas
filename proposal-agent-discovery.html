Session discovery: resume CLI-started sessions

If a user ran openhands from the terminal and later opens the canvas, we can discover and resume those sessions:

  1. Canvas queries each backend's GET /api/sessions (new endpoint)
  2. Backend returns ACP sessions started in the same cwd (workspace)
  3. Sessions not already tracked as canvas conversations appear as "resumable" in the sidebar
  4. Clicking one creates a new canvas conversation that attaches to the existing ACP session

4 SDK-Side Requirements

These are the additions needed in software-agent-sdk to make discovery work.

# What Where Details
1 SDK GET /api/agents endpoint agent-server HTTP router Returns an array of agent descriptors registered with this server.
Response: [{ id, name, version, description, icon_url?, capabilities }]
Initially returns exactly one entry (the OpenHands agent). Extensible for multi-agent servers later.
2 SDK AgentDescriptor type openhands/sdk/types.py Pydantic model:
id: str — unique agent ID (e.g. "openhands")
name: str — display name
version: str — semver
description: str | None
icon_url: str | None — optional icon
capabilities: AgentCapabilities
3 SDK AgentCapabilities type openhands/sdk/types.py list_sessions: bool = False — can the agent list existing sessions?
resume_session: bool = False — can the agent resume a paused session?
file_access: bool = True — does the agent read/write the workspace?
browser_tools: bool = False — does the agent support browser actions?
4 SDK GET /api/sessions endpoint agent-server HTTP router Optional — only available if capabilities.list_sessions = true.
Query params: ?cwd=/path — filter sessions by workspace dir.
Response: [{ session_id, agent_id, status, title?, created_at, updated_at }]
This wraps the ACP list_sessions(cwd) call that already exists but is never used.
5 SDK POST /api/sessions/{id}/resume endpoint agent-server HTTP router Optional — only available if capabilities.resume_session = true.
Resumes a paused ACP session. Canvas creates a conversation that wraps this session.
This wraps the ACP resume_session(cwd, id) call that already exists but is never used.
6 SDK Extend ServerInfo /server_info response Add optional agents_url: str | None field pointing to the /api/agents endpoint.
This lets the frontend know at health-check time whether agent discovery is available, before making extra calls.
# What Where Details
7 Frontend AgentDescriptor TypeScript type @openhands/typescript-client Mirror of the Python type. Added to types/base.ts and exported.
interface AgentDescriptor { id: string; name: string; version: string; description?: string; icon_url?: string; capabilities: AgentCapabilities; }
8 Frontend ServerClient.getAgents() @openhands/typescript-client async getAgents(): Promise<AgentDescriptor[]>
Calls GET /api/agents. Falls back to [] on 404 for older servers.
9 Frontend ServerClient.getSessions(cwd) @openhands/typescript-client async getSessions(cwd: string): Promise<SessionInfo[]>
Calls GET /api/sessions?cwd=…. Only called if the agent reports list_sessions: true.
10 Frontend useDiscoveredAgents hook agent-canvas React Query hook that polls all connected backends' /api/agents.
Merges results, deduplicates by agent ID, and annotates each entry with its source backend.
Replaces the static AGENT_OPTIONS array; queries backends selected in Step 0.
11 Frontend useDiscoverableSessions hook agent-canvas Background poll of /api/sessions for each backend that supports it.
Diffs against known conversations to surface "resumable" entries in sidebar.
12 Frontend Agent tag in conversation metadata agent-canvas Store agentId + backendId in ConversationMetadata when creating a conversation.
Display as colored tag in the conversation sidebar rows.

5 Implementation Phases

Phase A — Agent discovery SDK Frontend

  1. Add AgentDescriptor + AgentCapabilities types to SDK
  2. Implement GET /api/agents in agent-server (returns single OpenHands agent initially)
  3. Add optional agents_url to /server_info response
  4. Add AgentDescriptor type + getAgents() to typescript-client
  5. Create useDiscoveredAgents hook in agent-canvas
  6. Move Choose Backend to Step 0; replace static ChooseAgentStep with dynamic AvailableAgentsStep at Step 1

Phase B — Session discovery SDK Frontend

  1. Implement GET /api/sessions wrapping ACP list_sessions(cwd)
  2. Implement POST /api/sessions/{id}/resume wrapping ACP resume_session(cwd, id)
  3. Add getSessions() to typescript-client
  4. Create useDiscoverableSessions hook
  5. Add "resumable session" rows to conversation sidebar

Phase C — Agent tags Frontend

  1. Extend ConversationMetadata with agentId / backendId
  2. Store agent info when creating / resuming conversations
  3. Render agent tag in conversation sidebar rows
  4. Add agent filter to conversation search (stretch goal)

7 Open Questions

  1. Deduplication strategy: If Local and Docker both report the same agent (OpenHands), should the UI show them as one agent with two backends, or two separate entries? Current proposal: two entries, each tagged with its backend. Feedback welcome.
  2. Icon delivery: Should AgentDescriptor.icon_url be an absolute URL, a data URI, or an icon key that the frontend maps to a local asset? Leaning toward icon key for now (simpler, no CORS).
  3. Session resumption semantics: When the canvas creates a conversation wrapping an existing ACP session, should it replay the full event history or just show a "resumed from CLI" marker and start fresh? Leaning toward full replay for continuity.
  4. Polling vs. SSE for session discovery: Polling is simpler for v1. Worth adding Server-Sent Events later if we want instant detection of new CLI sessions.
  5. Auth for /api/agents and /api/sessions: Should these endpoints require the session API key? Probably yes — same auth as all other agent-server endpoints.

Summary

  • SDK adds 3 endpoints: /api/agents, /api/sessions, /api/sessions/{id}/resume
  • SDK adds 2 types: AgentDescriptor, AgentCapabilities
  • SDK extends: ServerInfo with optional agents_url
  • typescript-client adds: getAgents(), getSessions(), TypeScript mirrors of new types
  • agent-canvas adds: useDiscoveredAgents, useDiscoverableSessions hooks; reorders onboarding (backends → agents); conversation agent tags
  • 3 phases: (A) Agent discovery → (B) Session discovery → (C) Agent tags
